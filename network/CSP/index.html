Content Security Policy(CSP)

Content Security Policy(CSP)是帮助阻止或者最小化已知类型的网络安全威胁的风险的一个特性，CSP是由一系列网站向浏览器发出的指令组成的；这些指令指示浏览器对代码的可以做的事情进行限制。CSP的主要用例是用来控制哪些资源可以被浏览器加载（特别是JavaScript资源），使用CSP主要有下面这些目的：

  • 防御XSS攻击
  • 防御点击劫持攻击
  • 确认哪些网页可以在HTTPS之外加载

Content Security Policy的使用

开启CSP

启用CSP的方式有两种，针对单页面应用可以在页面中的<meta>标签中指定http-equiv属性开启；或者在请求的响应头中使用Content-Security-Policy进行指定

  • <mate>方式

    这种方式适合在单页面应用中进行使用（单页面只有一个html文件），并且该配置并不支持所有的CSP的特性（MDN中有说，后面会去探索哪些特性不支持）

    如果CSP中包含了default-src或者javascipt-src指令，行内javascript将不允许执行；除非额外地明确的允许行内javasciprt执行。可以通过unsafe-inline和unsafe-eval关键字来让browser忽略这个行为。
